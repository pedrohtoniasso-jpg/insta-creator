<objective>
Prove the reusable Instagram content system is reproducible, verifiable, and safe to reuse.

Purpose: validate the bootstrap and workflow in a clean environment, confirm deterministic outputs, and close the feedback loop for image and content review.
Output: reproducibility checks, verification evidence, approval-gate validation, and documentation update hooks that keep the system stable across future runs.
</objective>

<scope>
## In scope
- Verify deterministic folder names and repeatable storage outputs.
- Prove the bootstrap works in a clean environment.
- Confirm the expected artifacts exist after workflow execution.
- Validate approval gates for growth actions and reply batches.
- Verify generated images against the brief, cards, and project-spec visual rules.
- Capture rejection or revision feedback into project documentation.
- Add a human-readable verification report or summary artifact.
- Update `.planning/STATE.md` and `.planning/ROADMAP.md` if new durable findings are confirmed.

## Out of scope
- Redesigning the shared workflow core.
- Adding new brand behavior to the bootstrap.
- Reworking project-specific rules except where verification reveals a needed documentation update.
- Introducing new growth automation beyond the already-defined approval gates.
</scope>

<work_plan>
## Task 1 — Verify deterministic storage outputs
- Re-run the relevant workflow or storage checks in a clean or temporary environment.
- Confirm folder naming is repeatable and matches the agreed pattern.
- Verify the expected artifact set exists in each post folder.
- Check that traceability metadata stays attached to the bundle.

## Task 2 — Verify bootstrap reproducibility
- Run the bootstrap from a clean environment or equivalent isolated target.
- Confirm the bootstrap produces the same scaffold shape without manual intervention.
- Check that dry-run mode and validation behave as expected.
- Record any reproducibility quirks as durable documentation updates.

## Task 3 — Verify approval gates and growth behavior
- Confirm automatic growth actions remain separate from approval-gated replies.
- Validate that reply drafts are queued for batch approval before sending.
- Check that the audit trail preserves the decision history.
- Ensure the growth flow does not bypass the project-specific rules.

## Task 4 — Verify image and content alignment
- Compare generated images to the brief, cards, and project-spec visual rules.
- Capture evidence or notes describing whether the asset matches the requested direction.
- Record any rejection or revision feedback in the project documentation path.
- Keep the feedback loop explicit and human-readable.

## Task 5 — Produce final verification evidence
- Write a concise report summarizing the checks and outcomes.
- Record any durable findings in `.planning/STATE.md`.
- Keep `.planning/ROADMAP.md` aligned with the verified scope.
- Leave the phase ready for reuse or future project setup.
</work_plan>
