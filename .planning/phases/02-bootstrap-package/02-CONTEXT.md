# Phase 2: Bootstrap package - Context

**Gathered:** 2026-06-15
**Status:** Ready for planning

<domain>
## Phase Boundary
Create the reusable bootstrap for the shared Instagram content suite.

This phase defines how a new environment is prepared for the shared skill suite, how templates/defaults are installed, and how the bootstrap validates that setup. It does *not* introduce project-specific branding or content logic.
</domain>

<decisions>
## Implementation Decisions

### Bootstrap scope
- The bootstrap belongs to the shared skill suite.
- The bootstrap must stay generic and reusable across projects.
- Project branding and project-specific rules must remain external to the bootstrap.
- A modular bootstrap is preferred: a minimal core with optional add-ons for templates, validation, and guided onboarding.

### Onboarding shape
- The onboarding flow should be hybrid.
- Prefer a single command with sensible defaults.
- Ask follow-up questions only when required information is missing.
- If a project spec is already present, the bootstrap should consume it instead of recreating project decisions.

### Templates and defaults
- The bootstrap should generate the essential structural files only.
- Templates should be minimal but extensible.
- Defaults should make a new environment usable quickly without baking in brand-specific content.

### Validation and reproducibility
- Validation should be strong rather than superficial.
- Include a dry-run mode so users can preview what the bootstrap will do.
- Validation must confirm the scaffold is internally consistent and ready for downstream workflow skills.

### Coordination with phase 1 contracts
- The bootstrap must respect the shared-vs-project boundary defined in phase 1.
- The bootstrap must not hardcode brand rules.
- The bootstrap should prepare the environment so later workflow skills can read the project spec and storage contracts without modification.
</decisions>

<specifics>
## Specific Ideas
- The bootstrap should be easy to reuse for other brands beyond the current project.
- The bootstrap should create or validate only shared scaffolding, not content output.
</specifics>
