<objective>
Implement the reusable workflow skills that turn an idea into a content package.

Purpose: provide a deterministic shared workflow that starts from a brief, fans out into cards and caption generation, then assembles an approval-ready package with traceability preserved.
Output: a nucleus-plus-subskills skill suite with structured cards JSON, predictable captions, and a complete approval bundle stored in a deterministic post folder.
</objective>

<scope>
## In scope
- Define the orchestrating skill and its subskill boundaries.
- Implement the brief-generation skill.
- Implement structured cards JSON generation and validation.
- Implement the caption-generation skill with predictable structure and controlled variation.
- Implement the approval-package assembly skill.
- Persist all artifacts in a deterministic per-post folder.
- Add documentation and examples for the workflow outputs.
- Update `.planning/STATE.md` and `.planning/ROADMAP.md` if new stable workflow decisions need to be recorded.

## Out of scope
- Project-specific brand rules.
- Cron idea intake and project-spec loading.
- Growth automation outside approval packaging.
- Bootstrap package changes unless a workflow dependency is discovered.
- Full publishing integration if it is not required to assemble the content package.
</scope>

<work_plan>
## Task 1 — Define the workflow skill architecture
- Decide the orchestrator skill boundary and how it dispatches to subskills.
- Keep brief, cards, caption, and approval responsibilities separate.
- Define the minimum shared contract between the skills so the flow remains deterministic.
- Keep extension points explicit so future brands can swap only the pieces they need.

## Task 2 — Implement brief, cards, and caption generation
- Build the brief-generation skill as the first step in the flow.
- Implement cards generation as structured JSON with validation.
- Implement caption generation with a stable structure and allowed stylistic variation.
- Preserve traceability fields across the generated outputs.
- Ensure cards and caption generation can happen in parallel after the brief is ready.

## Task 3 — Implement approval packaging and storage flow
- Assemble the approval package only after the content artifacts are ready.
- Include the full review bundle needed to approve the post in one place.
- Include reply drafts when they are relevant to the package.
- Store the bundle in the deterministic per-post folder defined by the shared storage contract.
- Keep artifact naming and folder layout predictable and auditable.

## Task 4 — Validate the workflow outputs
- Validate the cards JSON schema and required fields.
- Verify the approval package includes the required artifacts and traceability metadata.
- Check that the folder structure remains deterministic and compatible with the storage contract.
- Confirm the workflow remains generic and does not hardcode project-specific branding.

## Task 5 — Capture reusable operational details
- Record any stable workflow decisions in `.planning/STATE.md`.
- Keep `.planning/ROADMAP.md` aligned with the implemented boundaries and outputs.
- Add workflow usage notes or examples that will help later phases and future projects.
</work_plan>
