# Carousel Storytelling Patterns

This reference captures the generic pattern that works best for clarity-first Instagram carousels in the Insta Creator workflow.

## Core idea
A carousel should feel like one continuous, swipeable value asset. Each slide should move the reader forward instead of acting like an isolated mini-post.

The target engagement behaviors are:
- swipes: the cover and each slide create curiosity for the next slide;
- saves: the deck contains a reusable rule, checklist, comparison, framework, or reminder;
- shares: the deck helps the viewer explain something to another person;
- comments/DMs: the CTA asks one natural question or invites private follow-up.

## Common structure
1. Hook / cover: specific promise, warning, comparison, or useful outcome.
2. Problem / stakes: what people get wrong and why it matters.
3. Context or mechanism: what is actually happening.
4. Practical insight, example, or reframing.
5. Saveable rule, checklist, or takeaway.
6. CTA: one relevant action — save, share, comment, or DM.

Shorter versions are fine when the topic does not need all six slides. Longer versions are acceptable only when every extra slide adds new value.

## Hook patterns
- Specific promise: “Como escolher X sem cair em Y”.
- Mistake warning: “O erro que faz você gastar mais com X”.
- Contrarian truth: “Nem sempre X é a melhor escolha”.
- Checklist/save cue: “Salva isso antes de fazer X”.
- Comparison tension: “X ou Y: quando cada um faz sentido”.
- Curiosity gap without clickbait: “O detalhe que muda sua decisão sobre X”.
- Audience mirror: “Se você faz X, olha isso antes de Y”.

## Carousel type patterns
- `checklist`: saveable steps.
- `mistakes`: error → correction.
- `myth_fact`: misconception → truth → action.
- `comparison`: option A vs B.
- `framework`: named rule or decision model.
- `how_to`: practical tutorial.
- `case_breakdown`: concrete example → lesson.
- `trend_explainer`: current signal → why it matters → action.
- `objection_handler`: doubt → answer → next step.
- `resource_list`: references worth saving.

## Writing rules
- Keep one main idea per slide.
- Use short phrases and avoid dense paragraphs.
- Make the first slide immediately clear or intriguing.
- Let the middle slides add logic, examples, or tension.
- End with a CTA that feels like the conclusion of the same argument.
- Prefer more simple slides over fewer crowded ones.
- Avoid vague covers like “Dicas sobre X” unless the slide also promises a concrete outcome.

## CTA rules
Choose one primary CTA:
- Save: checklists, frameworks, routines, decision rules.
- Share: warnings, reminders, comparisons, audience identity moments.
- Comment: preference, experience, objection discovery.
- DM: private/commercial/sensitive follow-up.

Do not stack several CTAs. Do not ask for meaningless comments.

## Visual rules
- Minimal, editorial, mobile-first.
- Large type.
- Plenty of negative space.
- Low-distraction backgrounds.
- Logo treatment should be stable and consistent across the set.
- No orphan/single-word wrapped lines.
